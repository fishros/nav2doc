导航插件

有许多插件接口供用户创建自己的自定义应用程序或算法。即costmap层、planner、控制器、行为树和恢复插件。下面列出了所有已知插件的列表，用于ROS 2导航。如果你知道一个插件，或者你已经创建了一个新的插件，请考虑提交一个包含该信息的请求。

可以在 sphinx_docs/plugins/index.rst 下找到并编辑此文件。有关创建自己的插件的教程，请参阅 编写新的代价地图2D插件, 编写新的代价地图2D插件、 编写新的控制器插件 、 编写新的规划器插件 或 编写新的恢复插件 。

代价地图层

插件名称

创造者

描述

Voxel Layer

埃坦-马尔德-埃普斯坦

使用深度和激光传感器读数和光线投射来清除自由空间的主像素层具有持久的3D体素层

Range Layer

大卫·卢

使用概率模型将来自在costmap上发布范围msgs的传感器的数据

Static Layer

埃坦-马尔德-埃普斯坦

获取静态 map 并将占用信息加载到costmap中

Inflation Layer

埃坦-马尔德-埃普斯坦

用指数衰减膨胀costmap中的致命障碍

Obstacle Layer

埃坦-马尔德-埃普斯坦

主拥有持久的2D costmap，从带有光线投射的2D激光扫描到清除自由空间

Spatio-Temporal Voxel Layer

史蒂夫·麦肯斯基

主传感器模型通过衰减获得时间3D稀疏体积体素网格

Non-Persistent Voxel Layer

史蒂夫·麦肯斯基

主属性3D占用网格仅包含最多的一组测量值

插件名称

Alexey Merzlyakov

控制器

插件名称

创造者

描述

动力传动系统支持

DWB Controller

大卫·卢!!

具有插件接口的高度可配置的DWA实现

差动、全向、有腿

TEB Controller

克里斯托夫·罗斯曼

适用于阿克曼、差分和完整机器人的类似MPC的控制器。

阿克曼，腿，全向微分

Regulated Pure Pursuit

史蒂夫·麦肯斯基

具有自适应特征的纯追踪算法的服务/工业机器人变体。

阿克曼，腿，差

Rotation Shim Controller

史蒂夫·麦肯斯基

在通过主控制器进行跟踪之前，一个 "shim" 控制器旋转到路径方向。

差速器，全向，模型旋转到位

规划器

插件名称

创造者

描述

动力传动系统支持

NavFn Planner

Eitan Marder-Eppstein & Kurt Konolige

使用A*或Dijkstra膨胀的导航函数，假设2D完整粒子

差动、全向、有腿
SmacPlannerHybrid

(原 SmacPlanner )

史蒂夫·麦肯斯基

使用杜宾或里德-谢普运动模型的SE2混合-A*实现，具有平滑和多分辨率查询。汽车、类似汽车和阿克曼汽车。运动学上可行。

阿克曼，差速器，全向，有腿

SmacPlanner2D

史蒂夫·麦肯斯基

使用4或8个具有平滑和多分辨率查询的连接邻域的2DA*实现

差动、全向、有腿

SmacPlannerLattice

史蒂夫·麦肯斯基

使用预先生成的最小控制集实现国家格规划器，用于可想象的任何类型的车辆的运动学可行规划。包括阿克曼、diff、omni和有腿机器人的生成器脚本。

微分，全向，阿克曼，腿，任意/定制

ThetaStarPlanner

Anshumaan辛格

使用4或8个连接的社区实现 θ *，假设机器人是2D完整粒子

微分，全向
