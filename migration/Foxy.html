Foxy转Galactic [校准@fish]

从ROS 2 Foxy到Galactic，增加了一些稳定性改进，我们在这里不会具体讨论。 [校准@fish]

导航Action反馈更新 [校准@fish]

导航操作反馈有两个改进: [校准@fish]

  • 这个``distance_remaining`` now集成路径对报告准确剩余距离去。以前，此字段报告欧氏距离当前姿势目标构成。 [校准@fish]

  • 添加 estimated_time_remaining 领域。此字段报告估计剩余时间除以剩余距离当前速度。 [校准@fish]

NavigateToPose BT节点接口变化 [校准@fish]

NavigateToPose输入端口已更改为PoseStamped，而不是点和四元数。 [校准@fish]

有关更多信息，请参见 NavigateToPose [校准@fish] 。 [校准@fish]

NavigateThroughPoses和ComputePathThroughPosesActions添加 [校准@fish]

这个 NavigateThroughPoses 作用已被添加到 NavigateToPose 类似物中。这个Action将允许用户在开始姿势和最终姿势之间指定一些硬的中间姿势约束来计划，而不是去一个单一的位置。新的 ComputePathThroughPoses 行动已加入 planner_server ，通过 N goal_poses 处理这些请求。 [校准@fish]

工具``ComputePathThroughPoses`` 动作服务器将一套 N 目标，计划每个姿势并连接输出路径用于导航。控制器和导航仪一无所知语义的生成路径，机器人不会停止或慢速方法这些目标。它将继续通过每个姿势好像任何其他点路径不断。当搭配 SmacPlanner ，此功能可用于生成完全运动学可行轨迹通过构成约束。 [校准@fish]

如果你希望在每个目标姿势停止，考虑使用航点跟随者，这将停止并允许用户在每个姿势选择执行任务插件。 [校准@fish]

ComputePathToPose BT节点接口变化 [校准@fish]

添加了 start 输入端口，以可选地允许请求从 start 到 goal 的路径，而不是从机器人的当前位置到 goal 的路径。 [校准@fish]

有关更多信息，请参见 ComputePathToPose [校准@fish] 。 [校准@fish]

ComputePathToPoseAction界面变化 [校准@fish]

  • 球门姿势场 pose 改为 goal 。 [校准@fish]

  • 已经增加了 start 装填区。 [校准@fish]

  • bool字段 use_start 已添加。 [校准@fish]

当 use_start 为真时，添加这两个附加字段是为了可选地允许从 start 到 goal 的路径请求，而不是从机器人的当前位置到 goal 的路径。已经对规划器服务器进行了相应的更改。 [校准@fish]

备份BT节点接口变化

这个 backup_dist 和 backup_speed 的输入端口都应该是正值，分别表示机器人向后行驶的距离。 [校准@fish]

备份恢复界面变化

这个``speed`` in a backup recovery goal should be positive indicating the speed with which to drive backward. target.x in的备份恢复目标应该是正的，表示向后行驶的距离。在这两种情况下，负值都被无声地反转。 [校准@fish]

这个Nav2控制器和目标检查插件接口变化 [校准@fish]

新概念与空间依赖物体 "Costmap Filters" 出现Galactic (有关此概念的更多信息可以发现在 导航相关概念 页面)。代价地图滤波器作为costmap插件，应用于单独costmap以上普通插件。为了制作过滤后的成本图并改变机器人在注释区域的行为，过滤器插件读取来自过滤器蒙版的数据。然后数据线性转化特征地图过滤器空间。它可以通行的区域，最大速度限制m/s，机器人期望方向度或别的。转化特征映射跟着地图/costmap，传感器数据和当前机器人位置用于插件's algorithms to make required updates in the resulting costmap and robot's行为。 [校准@fish]

每个costmap过滤器订阅过滤器信息主题 (由 Costmap Filter Info Publisher Server 发布)，其中包含加载的costmap过滤器和过滤器掩码主题的所有必要信息。 SpeedFilter 另外发布了针对控制器的最大速度限制 messages ，以强制机器人不会超过给定的限制。 [校准@fish]

代价地图2D current_ 的使用 [校准@fish]

在costmap2D中，ROS1中使用了 current_ ，以表示costmap层是否仍处于启用状态并积极处理数据。只有在传感器的预期更新率没有达到的情况下，它才会变成 false ，因此它变得陈旧或没有消息。它充当导航传感器停止发布时的自动防故障装置。 [校准@fish]

在银河，这将继续，但它也将增加额外的能力。当成本图由于清理或其他导航恢复而重置时，它现在也被设置为 false 。这将阻止机器人创建计划或控制工作，直到成本图在重置后至少更新一次。这使我们能够确保我们永远不会使用完全空的costmap创建路径或控件，这可能会导致冲突，因为清除costmap并立即请求运行算法。 [校准@fish]
