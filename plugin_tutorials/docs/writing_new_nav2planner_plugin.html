编写新的规划器插件

  • Overview

  • Requirements

  • Tutorial Steps

概述

本教程展示了如何创建自己的planner插件。 [校准@fish]

要求

  • ROS 2 (二进制或从源代码构建) [校准@fish]

  • Nav2 (包括依赖项) [校准@fish]

  • Gazebo

  • Turtlebot3

教程步骤

1创建新规划器插件

我们将创建一条简单的直线planner。本教程中的注释代码可以在 navigation_tutorials 库中找到，因为 nav2_straightline_planner 这个包可以被认为是编写planner插件的参考。 [校准@fish]

我们的示例插件继承自基类 “nav2_core::Global规划器”。基类提供5种纯虚拟方法来实现planner插件。该插件将被planner服务器用来计算轨迹。让我们进一步了解编写planner插件所需的方法。 [校准@fish]

虚拟方法

方法描述

Requires需要覆盖？** [校准@fish]

configure() [校准@fish]

方法时planner服务器进入on_configure状态。理想这方法执行声明ROS参数初始化planner的成员变量。该方法4输入参数: 共享指针父节点，planner名称，tf缓冲区指针共享指针costmap。 [校准@fish]

是的

activate()

方法当planner服务器进入on_activate状态。理想此方法执行的操作必要之前planner去活动状态。 [校准@fish]

是的

deactivate() [校准@fish]

方法在planner服务器进入on_deactivate状态时调用。理想情况下，这种方法应该在planner进入非活动状态之前实施必要的操作。 [校准@fish]

是的

cleanup()

方法在planner服务器进入on_clean状态时调用。理想情况下，这种方法应该清理为planner创造的资源。 [校准@fish]

是的

createPlan() [校准@fish]

当planner服务器要求为指定的开始姿势和目标姿势制定全局计划时，调用方法。此方法返回携带全局计划的 “nav_msgs::msg:: path”。该方法需要2个输入参数: 开始姿势和目标姿势。 [校准@fish]

是的

对于本教程，我们将使用方法 StraightLine::configure() 和 StraightLine::createPlan() 创建直线规划器。 [校准@fish]

在planner， configure() 方法必须员变量ROS参数和任何初始化， [校准@fish]

node_ = parent;
tf_ = tf;
name_ = name;
costmap_ = costmap_ros->getCostmap();
global_frame_ = costmap_ros->getGlobalFrameID();

// Parameter initialization
nav2_util::declare_parameter_if_not_declared(node_, name_ + ".interpolation_resolution", rclcpp::ParameterValue(0.1));
node_->get_parameter(name_ + ".interpolation_resolution", interpolation_resolution_);

这里，''name _ + ”。interpolation_resolution ” 是获取ROS参数 interpolation_resolution 特定于我们planner。Navigation2允许加载多插件并保持组织每个插件映射到一些账号/名称。现在如果我们想检索参数特定插件，我们使用 <mapped_name_of_plugin>.<name_of_parameter> 作为完成上述片段。例如，示例planner映射名称 "GridBased" 并检索 interpolation_resolution 参数特定 "GridBased" ，我们使用 Gridbased.interpolation_resolution 。换句话说， GridBased 作为命名空间插件特定参数。当我们讨论参数文件 (或params文件) 时，我们将看到更多有关此的信息。 [校准@fish]

在 createPlan() 法中，我们需要创建一条从给定的开始到目标姿势的路径。使用开始姿态和目标姿态调用 “直线::createPlan()” 来解决全局路径规划问题。成功后，它将路径转换为 “nav_msgs::msg:: path” 并返回到planner服务器。下面的注释显示了此方法的实现。 [校准@fish]

nav_msgs::msg::Path global_path;

// Checking if the goal and start state is in the global frame
if (start.header.frame_id != global_frame_) {
  RCLCPP_ERROR(
    node_->get_logger(), "Planner will only except start position from %s frame",
    global_frame_.c_str());
  return global_path;
}

if (goal.header.frame_id != global_frame_) {
  RCLCPP_INFO(
    node_->get_logger(), "Planner will only except goal position from %s frame",
    global_frame_.c_str());
  return global_path;
}

global_path.poses.clear();
global_path.header.stamp = node_->now();
global_path.header.frame_id = global_frame_;
// calculating the number of loops for current value of interpolation_resolution_
int total_number_of_loop = std::hypot(
  goal.pose.position.x - start.pose.position.x,
  goal.pose.position.y - start.pose.position.y) /
  interpolation_resolution_;
double x_increment = (goal.pose.position.x - start.pose.position.x) / total_number_of_loop;
double y_increment = (goal.pose.position.y - start.pose.position.y) / total_number_of_loop;

for (int i = 0; i < total_number_of_loop; ++i) {
  geometry_msgs::msg::PoseStamped pose;
  pose.pose.position.x = start.pose.position.x + x_increment * i;
  pose.pose.position.y = start.pose.position.y + y_increment * i;
  pose.pose.position.z = 0.0;
  pose.pose.orientation.x = 0.0;
  pose.pose.orientation.y = 0.0;
  pose.pose.orientation.z = 0.0;
  pose.pose.orientation.w = 1.0;
  pose.header.stamp = node_->now();
  pose.header.frame_id = global_frame_;
  global_path.poses.push_back(pose);
}

global_path.poses.push_back(goal);

return global_path;

其余方法不使用但强制性覆盖它们。根据规则，我们覆盖所有但留在空白。 [校准@fish]

2-导出planner插件 [校准@fish]

既然我们已经创建了我们的自定义planner，我们需要导出我们的planner插件，以便它对planner服务器可见。插件在运行时加载，如果它们不可见，那么我们的planner服务器将无法加载它。在ROS 2中，导出和加载插件由 pluginlib 处理。 [校准@fish]

在我们的教程里，类 nav2_straightline_planner::StraightLine 被动态加载为 nav2_core::GlobalPlanner [校准@fish]

  1. 为了导出planner，我们需要提供两条生产线 [校准@fish]

#include "pluginlib/class_list_macros.hpp"
PLUGINLIB_EXPORT_CLASS(nav2_straightline_planner::StraightLine, nav2_core::GlobalPlanner)

请注意，它需要pluginlib导出插件的类。Pluginlib将提供作为宏的 PLUGINLIB_EXPORT_CLASS ，它负责所有的导出工作。 [校准@fish]

它最好地方这些诗句结束文件但技术，你也可以写顶部。 [校准@fish]

  1. 下一步是创建插件的说明文件根目录的包。例如，我们的教程包中的 global_planner_plugin.xml 文件。此文件包含以下信息 [校准@fish]

  • 参数``library path``:插件的库名和它的位置。 [校准@fish]

  • 参数``class name``:class的名称。 [校准@fish]

  • 参数``class type``:类的类型 [校准@fish]

  • 参数``base class``:基类的名称。 [校准@fish]

  • 参数``description``:插件的描述。 [校准@fish]

<library path="nav2_straightline_planner_plugin">
  <class name="nav2_straightline_planner/StraightLine" type="nav2_straightline_planner::StraightLine" base_class_type="nav2_core::GlobalPlanner">
    <description>This is an example plugin which produces straight path.</description>
  </class>
</library>

  1. 下一步是导出插件使用 CMakeLists.txt 使用cmake功能 pluginlib_export_plugin_description_file() 。此函数将插件描述文件安装到 share 目录，并设置属性索引以使其可被发现。 [校准@fish]

pluginlib_export_plugin_description_file(nav2_core global_planner_plugin.xml)

  1. 插件描述文件也应添加到 package.xml [校准@fish]

<export>
  <build_type>ament_cmake</build_type>
  <nav2_core plugin="${prefix}/global_planner_plugin.xml" />
</export>

  1. 编译并注册。接下来，我们将使用这个插件。 [校准@fish]

3通过插件名称通过params文件 [校准@fish]

要启用插件，我们需要将 nav2_params.yaml 文件修改为以下内容，以替换以下参数 [校准@fish]

注解

用Galactic或后， plugin_names 和 plugin_types 替换单个 plugins string载体插件名称。类型现在定义在 plugin_name 命名空间 ''plugin: `` field (e.g. `` 插件: MyPlugin: Plugin'')。内联注释代码块将帮助您通过这个。 [校准@fish]

planner_server:
ros__parameters:
  planner_plugin_types: ["nav2_navfn_planner/NavfnPlanner"] # For Eloquent and earlier
  planner_plugin_ids: ["GridBased"] # For Eloquent and earlier
  plugins: ["GridBased"] # For Foxy and later
  use_sim_time: True
  GridBased:
    plugin: nav2_navfn_planner/NavfnPlanner # For Foxy and later
    tolerance: 2.0
    use_astar: false
    allow_unknown: true

与

planner_server:
ros__parameters:
  planner_plugin_types: ["nav2_straightline_planner/StraightLine"] # For Eloquent and earlier
  planner_plugin_ids: ["GridBased"] # For Eloquent and earlier
  plugins: ["GridBased"] # For Foxy and later
  use_sim_time: True
  GridBased:
    plugin: nav2_straightline_planner/StraightLine # For Foxy and later
    interpolation_resolution: 0.1

在上述片段，你可以观察映射的 nav2_straightline_planner/StraightLine planner其账号 GridBased 。为了传递插件特定的参数，我们使用了 <plugin_id>.<plugin_specific_parameter> 。 [校准@fish]

4-运行直线插件

在启用导航的情况下运行Turtlebot3 模拟2。如何制作它的详细说明写在 入门 。下面是该命令的快捷方式: [校准@fish]

$ ros2 launch nav2_bringup tb3_simulation_launch.py params_file:=/path/to/your_params_file.yaml

然后转到RViz，点击顶部的 "2D Pose Estimate" 按钮，指出地图上的位置，就像 入门 中描述的那样。机器人将在地图上定位，然后点击 "Navigation2 goal" ，点击你想让你的planner考虑目标姿势的姿势。之后，planner将规划路径，机器人将开始向目标移动。 [校准@fish]
