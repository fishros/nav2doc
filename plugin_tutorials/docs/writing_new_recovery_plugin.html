编写新的恢复插件

  • Overview

  • Requirements

  • Tutorial Steps

概述

本教程演示如何创建自己的恢复插件。恢复插件位于恢复服务器中。与planner和控制器服务器不同，每个恢复都将托管自己独特的操作服务器。planner和控制器在完成相同任务时具有相同的API。但是，恢复可用于执行各种各样的任务，因此每个恢复都可以有自己独特的操作消息定义和服务器。这允许恢复服务器的巨大灵活性，实现任何可想象的不需要其他重用的恢复操作。

要求

  • ROS 2 (二进制或从源代码构建)

  • Nav2 (包括依赖项)

  • Gazebo

  • Turtlebot3

教程步骤

1-创建一个新的恢复插件

我们将创建一个简单呼救回收行为。它将使用Twilio发送消息通过短信远程操作中心。本教程中的代码在 navigation_tutorials 库 nav2_sms_recovery 。这个包可以参考写作回收插件。

我们例子插件实现插件的 nav2_core::Recovery。然而，我们愉快包装行动 nav2_recoveries ，所以我们使用 nav2_recoveries::Recovery 底座类应用程序。此包装类来自 nav2_core 类，可以作为一个插件，但处理绝大多数ROS 2动作服务器锅炉板需要。

基类 nav2_core 提供4纯虚拟方法执行恢复插件。插件将使用恢复服务器主机插件，但每个插件提供自己独特动作服务器接口。让学习更方法需要写恢复插件 * * 如果你没有使用 nav2_recoveries 包装 * *。

方法以恢复更新速率调用，并应完成任何必要的更新。旋转的一个例子是计算当前周期的命令速度，发布它并检查完成情况。

在本教程中，我们将使用方法 onRun() 、 onCycleUpdate() 和 onConfigure() 来创建SMS恢复。为了简洁起见，将跳过 onConfigure() ，但只包括参数声明。

在恢复中， onRun() 法必须设置任何初始状态并开始恢复行为。对于我们寻求帮助恢复行为的情况，我们可以用这种方法简单地计算我们所有的需求。

Status SMSRecovery::onRun(const std::shared_ptr<const Action::Goal> command)
{
  std::string response;
  bool message_success = _twilio->send_message(
    _to_number,
    _from_number,
    command->message,
    response,
    "",
    false);

  if (!message_success) {
    RCLCPP_INFO(node_->get_logger(), "SMS send failed.");
    return Status::FAILED;
  }

  RCLCPP_INFO(node_->get_logger(), "SMS sent successfully!");
  return Status::SUCCEEDED;
}

我们收到一个行动目标， command ，我们想在其中进行处理。 command 包含一个 message 字段，其中包含我们想要传达给我们母亲的信息。这是我们想通过短信发送给行动中心的战友的 "call for help" 信息。

我们使用服务Twilio完成此任务。请 create an account 和所有相关信息需要创建服务 (例如 account_sid ， auth_token ，电话号码)。你可以设置这些值参数配置文件对应 onConfigure() 参数声明。

我们使用 _twilio 对象从配置文件中发送包含您的帐户信息的消息。我们发送消息并登录以筛选消息是否发送成功。我们根据要返回给动作客户端的值返回 FAILED 或 SUCCEEDED 。

onCycleUpdate() is trivially simple as a result of our short-running recovery behavior. If the recovery was instead longer running like spinning, navigating to a safe area, or getting out of a bad spot and waiting for help, then this function would be checking for timeouts or computing control values. For our example, we simply return success because we already completed our mission in onRun().

Status SMSRecovery::onCycleUpdate()
{
  return Status::SUCCEEDED;
}

不使用其余方法，也不强制重写它们。

2-导出恢复插件

现在我们创建定制回收，我们需要导出恢复插件，可见到恢复服务器。插件加载运行时如果不可见，然后我们恢复服务器将无法加载。在ROS 2中，导出和加载插件由 pluginlib 处理。

来到我们的教程，类 “nav2_sms_recovery::SMSRecovery `` is loaded dynamically as `` nav2_core:: recovery” 是我们的基类。

  1. 导出恢复，我们需要提供两行

#include "pluginlib/class_list_macros.hpp"
PLUGINLIB_EXPORT_CLASS(nav2_sms_recovery::SMSRecovery, nav2_core::Recovery)

请注意，它需要pluginlib导出插件的类。Pluginlib将提供作为宏的 PLUGINLIB_EXPORT_CLASS ，它负责所有的导出工作。

它最好地方这些诗句结束文件但技术，你也可以写顶部。

  1. 下一步是在包的根目录中创建插件的描述文件。例如，我们的教程包中的 recovery_plugin.xml 文件。此文件包含以下信息

  • 参数``library path``:插件的库名和它的位置。

  • 参数``class name``:class的名称。

  • 参数``class type``:类的类型

  • 参数``base class``:基类的名称。

  • 参数``description``:插件的描述。

<library path="nav2_sms_recovery_plugin">
  <class name="nav2_sms_recovery/SMSRecovery" type="nav2_sms_recovery::SMSRecovery" base_class_type="nav2_core::Recovery">
    <description>This is an example plugin which produces an SMS text message recovery.</description>
  </class>
</library>

  1. 下一步是导出插件使用 CMakeLists.txt 使用cmake功能 pluginlib_export_plugin_description_file() 。此函数将插件描述文件安装到 share 目录，并设置属性索引以使其可被发现。

pluginlib_export_plugin_description_file(nav2_core recovery_plugin.xml)

  1. 插件描述文件也应添加到 package.xml

<export>
  <build_type>ament_cmake</build_type>
  <nav2_core plugin="${prefix}/recovery_plugin.xml" />
</export>

  1. 编译并注册。接下来，我们将使用这个插件。

4运行恢复插件

运行Turtlebot3 仿真启用Nav2。如何制作它的详细说明写在 入门 。下面是该命令的快捷方式:

$ ros2 launch nav2_bringup tb3_simulation_launch.py params_file:=/path/to/your_params_file.yaml

在终端运行:
