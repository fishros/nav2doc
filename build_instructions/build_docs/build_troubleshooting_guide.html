构建故障排除指南

常见的Nav2依赖项构建失败 [校准@fish]

  • 确保。bashrc文件中没有ROS环境变量。打开新的终端，然后尝试重新构建包。 [校准@fish]

  • 确保为正确的ROS 2发行版运行rosdep。 rosdep install -y -r -q --from-paths src --ignore-src --rosdistro <ros2-distro> [校准@fish]

  • 确保 setup.bash 来自ROS 2安装或ROS 2主构建工作区 (如果适用)。检查是否可以运行talker和listen节点。 [校准@fish]

  • 确保 nav2_depend_ws/install 中的 setup.bash 来源。 [校准@fish]

  • 检查您是否有正确的ROS版本和发行版。项目完结，系统自动填充内容 [校准@fish]

  • 如果你在启动时发现一堆关于 map 或 odom 框架不存在的错误，记得激活驱动程序 (或用于模拟的露台) 并在 map 框架中设置一个初始姿势。代价地图2D将阻止激活，直到有一个完整的TF树可用。 [校准@fish]

  • 如果你没有看到转换或服务器运行，请确保你已经激活了生命周期节点。 [校准@fish]

  • 搜索 GitHub Issues [校准@fish]

  • 确保您为您的分配使用了正确的分支。没有十字架支持分支 DistroA 在 DistroB 。主要开发分支使用滚动分布。 [校准@fish]

还是解决不了？让我们知道您的问题。 Open a ticket 。 [校准@fish]

报告问题

  • 如果您在构建导航2时遇到任何问题，您可以使用 GitHub and always feel free to open a ticket 问题选项卡中的搜索工具。 [校准@fish]
