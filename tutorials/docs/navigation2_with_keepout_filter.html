使用禁止导航区

  • Overview

  • Requirements

  • Tutorial Steps

概述

本教程演示如何简单地利用保持/安全区机器人不能进入首选车道机器人移动工业环境和仓库。这些功能是由 KeepoutFilter costmap过滤器插件将启用本文件。 [校准@fish]

要求

假设ROS 2、Gazebo和TurtleBot3包是在本地安装或构建的。请确保Nav2项目也是在当地建造的，因为它是在 构建和安装 制造的。 [校准@fish]

教程步骤

1.准备滤镜蒙版

正如 导航相关概念 所写，任何代价地图过滤器 (包括Keepout过滤器) 都在读取过滤器掩码文件中标记的数据。过滤器掩码-是通常通过PGM、PNG或BMP光栅文件分发的Nav22d地图，其元数据包含在YAML文件中。以下步骤有助于了解如何制作新的过滤器蒙版: [校准@fish]

创建具有PGM/PNG/BMP格式的新图像: 将将用于世界模拟的 turtlebot3_world.pgm 主地图从 Nav2 存储库复制到新的 keepout_mask.pgm 文件中。 [校准@fish]

在您最喜欢的光栅图形编辑器中打开 keepout_mask.pgm (例如，可以采用GIMP编辑器)。蒙版上每个像素的亮度意味着您要使用的特定costmap过滤器的编码信息。每个像素的颜色亮度属于 [0..255] 范围 (或 [0..100] 百分比范围)，其中 0 的意思是黑色和 255 白色。另一个术语 "darkness" 将被理解为与轻盈完全相反。换句话说，“颜色 _ 黑暗 = 100%-颜色 _ 亮度”。 [校准@fish]

在GIMP中，亮度通过颜色分量值 (例如， R 百分比) 来表示，并且可以通过在变色工具中移动 L 滑块来设置: [校准@fish]

传入面具文件读取地图服务器转换成 OccupancyGrid 值 [0..100] 范围 ( 0 手段游离细胞， 100 占领，任何间或多或少占领细胞地图) 或等于 -1 未知值。在Nav2栈每个地图 mode 属性可 trinary ， scale 或 raw 。根据 mode 选择，色彩明度PGM/PNG/BMP被转换为 OccupancyGrid 的以下原则: [校准@fish]

  • trinary (default mode): Darkness >= occupied_thresh means that map occupied (100). Darkness <= free_thresh - map free (0). Anything in between - unknown status on map (-1). [校准@fish]

  • scale: Alpha < 1.0 - unknown. Darkness >= occupied_thresh means that map occupied (100). Darkness <= free_thresh - map free (0). Anything in between - linearly interpolate to nearest integer from [0..100] range. [校准@fish]

  • raw: Lightness = 0 (dark color) means that map is free (0). Lightness = 100 (in absolute value) - map is occupied (100). Anything in between - OccupancyGrid value = lightness. Lightness >= 101 - unknown (-1). [校准@fish]

其中 free_thresh 和 occupied_thresh 阈值以最大亮度/黑暗水平 ( 255 ) 的百分比表示。映射模式和阈值作为 mode 、 free_thresh 和 occupied_thresh 字段放置在YAML元数据文件 (见下文) 中。 [校准@fish]

注解

YAML元数据文件中有另一个参数，称为 negate 。默认情况下，它设置为 false 。当它被设置为 true 时，黑色像素将被认为是自由的，白色像素-被占用。在这种情况下，我们应该计算 trinary 和 scale 模式的颜色亮度而不是黑暗。 negate 对 raw 模式没有影响。 [校准@fish]

用禁止过滤器 OccupancyGrid 价值成正比更大可能的区域corresponting这个电池: 高等值意味着不可逾越的地区。细胞占领值覆盖保持的地区机器人永远不会进入或通过。 KeepoutFilter 也可以作为 "weighted areas layer" 设置 OccupancyGrid 东西之间 [1-99] 非占领值。机器人是允许在这些地区，然而其存在将 "undesirable" (值越高，越早planners将试图获取机器人的区域)。 [校准@fish]

Keepout过滤器还包括首选车道的情况，在这种情况下，机器人应该只在预定义的车道和允许的区域移动，例如在仓库中。要使用这种羽毛，您需要准备蒙版图像，其中车道和允许区域将标有自由值，而所有其他区域将被占用。在 trinary 或 scale 模式下绘制遮罩的提示: 通常，属于车道的像素量远远少于覆盖其他区域的像素量。在这种情况下，最初所有的车道数据都可以用黑色铅笔在白色背景上绘制，然后 (就在保存PGM之前) 可以使用图像光栅编辑器中的 "color inversion" 工具。 [校准@fish]

为简单起见，在示例中用黑色填充区域 (在 trinary 模式下，这意味着被占用的地图)，您将其标记为禁区: [校准@fish]

在所有保留区域被填满后，保存 keepout_mask.pgm 图像。 [校准@fish]

像所有其他映射一样，过滤器掩码应该有自己的YAML元数据文件。将 turtlebot3_world.yaml 复制到 keepout_mask.yaml 。打开 keepout_mask.yaml 并纠正 image 领域的一个新的PGM面具: [校准@fish]

image: turtlebot3_world.pgm
->
image: keepout_mask.pgm

由于过滤器蒙版图像是作为主映射的副本创建的，因此不需要更改YAML文件的其他字段。保存 keepout_mask.yaml ，新的过滤面罩就可以使用了。 [校准@fish]

注解

世界地图本身和过滤蒙版可以有不同的大小，可能有用的原点和分辨率，例如，当过滤器蒙版覆盖地图上较小的区域时，或者当一个过滤器蒙版被重复使用多次时 (例如，在酒店)。在这种情况下，您还需要更正YAML中的 resolution 和 origin 字段，以便将过滤器掩码正确放置在原始地图的顶部。 [校准@fish]

注解

另外很重要的一点是因为代价地图2D不支持定位，最后第三 "yaw" 组件的 origin 矢量应等于零。例如: ''origin: [1.25-5.18，0.0]。 [校准@fish]

2.配置代价地图过滤器信息发布服务器 [校准@fish]

每个costmap过滤器读取 nav2_msgs/CostmapFilterInfo 类型消息中的传入元信息 (例如过滤器类型或数据转换系数)。这些信息由 Costmap Filter Info Publisher Server 出版。服务器作为生命周期节点运行。根据 design document 的说法， nav2_msgs/CostmapFilterInfo 的消息与 OccupancyGrid 的过滤面具主题成对。因此，与代价地图过滤器信息发布服务器一起，应该启用配置为发布过滤器掩码的地图服务器的新实例。 [校准@fish]

为了在配置中启用Keepout筛选器，应在Python启动文件中将两个服务器作为生命周期节点启用。例如，这可能如下所示: [校准@fish]

import os

from ament_index_python.packages import get_package_share_directory

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument
from launch.substitutions import LaunchConfiguration
from launch_ros.actions import Node
from nav2_common.launch import RewrittenYaml


def generate_launch_description():
    # Get the launch directory
    costmap_filters_demo_dir = get_package_share_directory('nav2_costmap_filters_demo')

    # Create our own temporary YAML files that include substitutions
    lifecycle_nodes = ['filter_mask_server', 'costmap_filter_info_server']

    # Parameters
    namespace = LaunchConfiguration('namespace')
    use_sim_time = LaunchConfiguration('use_sim_time')
    autostart = LaunchConfiguration('autostart')
    params_file = LaunchConfiguration('params_file')
    mask_yaml_file = LaunchConfiguration('mask')

    # Declare the launch arguments
    declare_namespace_cmd = DeclareLaunchArgument(
        'namespace',
        default_value='',
        description='Top-level namespace')

    declare_use_sim_time_cmd = DeclareLaunchArgument(
        'use_sim_time',
        default_value='true',
        description='Use simulation (Gazebo) clock if true')

    declare_autostart_cmd = DeclareLaunchArgument(
        'autostart', default_value='true',
        description='Automatically startup the nav2 stack')

    declare_params_file_cmd = DeclareLaunchArgument(
            'params_file',
            default_value=os.path.join(costmap_filters_demo_dir, 'params', 'keepout_params.yaml'),
            description='Full path to the ROS 2 parameters file to use')

    declare_mask_yaml_file_cmd = DeclareLaunchArgument(
            'mask',
            default_value=os.path.join(costmap_filters_demo_dir, 'maps', 'keepout_mask.yaml'),
            description='Full path to filter mask yaml file to load')

    # Make re-written yaml
    param_substitutions = {
        'use_sim_time': use_sim_time,
        'yaml_filename': mask_yaml_file}

    configured_params = RewrittenYaml(
        source_file=params_file,
        root_key=namespace,
        param_rewrites=param_substitutions,
        convert_types=True)

    # Nodes launching commands
    start_lifecycle_manager_cmd = Node(
            package='nav2_lifecycle_manager',
            executable='lifecycle_manager',
            name='lifecycle_manager_costmap_filters',
            namespace=namespace,
            output='screen',
            emulate_tty=True,  # https://github.com/ros2/launch/issues/188
            parameters=[{'use_sim_time': use_sim_time},
                        {'autostart': autostart},
                        {'node_names': lifecycle_nodes}])

    start_map_server_cmd = Node(
            package='nav2_map_server',
            executable='map_server',
            name='filter_mask_server',
            namespace=namespace,
            output='screen',
            emulate_tty=True,  # https://github.com/ros2/launch/issues/188
            parameters=[configured_params])

    start_costmap_filter_info_server_cmd = Node(
            package='nav2_map_server',
            executable='costmap_filter_info_server',
            name='costmap_filter_info_server',
            namespace=namespace,
            output='screen',
            emulate_tty=True,  # https://github.com/ros2/launch/issues/188
            parameters=[configured_params])

    ld = LaunchDescription()

    ld.add_action(declare_namespace_cmd)
    ld.add_action(declare_use_sim_time_cmd)
    ld.add_action(declare_autostart_cmd)
    ld.add_action(declare_params_file_cmd)
    ld.add_action(declare_mask_yaml_file_cmd)

    ld.add_action(start_lifecycle_manager_cmd)
    ld.add_action(start_map_server_cmd)
    ld.add_action(start_costmap_filter_info_server_cmd)

    return ld

在 params_file 变量应设置YAML文件有ROS参数代价地图过滤器信息发布服务器和地图服务器节点。这些参数及其含义列在 地图服务器/保护程序 页面。请参考信息。为例 params_file 可以发现如下: [校准@fish]

costmap_filter_info_server:
  ros__parameters:
    use_sim_time: true
    type: 0
    filter_info_topic: "/costmap_filter_info"
    mask_topic: "/keepout_filter_mask"
    base: 0.0
    multiplier: 1.0
filter_mask_server:
  ros__parameters:
    use_sim_time: true
    frame_id: "map"
    topic_name: "/keepout_filter_mask"
    yaml_filename: "keepout_mask.yaml"

请注意:

  • 对于保持过滤器，costmap过滤器的 type 应设置为 0 。 [校准@fish]

  • 过滤器面具主题名称应平等的 mask_topic 参数代价地图过滤器信息发布服务器和 topic_name 参数地图服务器。 [校准@fish]

  • 根据代价地图过滤器设计，在过滤器空间中， OccupancyGrid 值被线性转换为特征图。对于Keepout筛选器，这些值直接作为筛选器空间值传递，而无需进行线性转换。即使 base 和 multiplier 系数没有用于保持过滤器，它们应该被设置为 0.0 ，并相应地 1.0 ，以明确地表明我们有从 OccupancyGrid 值-> 到过滤器值空间的一对一转换。 [校准@fish]

准备去独立Python启动脚本，YAML文件ROS参数过滤器掩码示例用禁止过滤器可以发现 nav2_costmap_filters_demo 目录 navigation2_tutorials 库。简单运行过滤器信息发布服务器和地图服务器调整Turtlebot3 标准仿真写在 入门 ，构建演示Launch costmap_filter_info.launch.py 如下: [校准@fish]

$ mkdir -p ~/tutorials_ws/src
$ cd ~/tutorials_ws/src
$ git clone https://github.com/ros-planning/navigation2_tutorials.git
$ cd ~/tutorials_ws
$ colcon build --symlink-install --packages-select nav2_costmap_filters_demo

3.启用保留过滤器

代价地图过滤器是Costamp2D插件。您可以启用 KeepoutFilter 插件代价地图2D添加 keepout_filter 的 plugins 参数 nav2_params.yaml 。你可以把它放到 global_costmap 规划与禁止区和 local_costmap 确保机器人不会试图通过保留区。的KeepoutFilter插件应该以下参数定义: [校准@fish]

  • plugin: type of plugin. In our case nav2_costmap_2d::KeepoutFilter. [校准@fish]

  • 上一章中的代价地图过滤器信息发布者服务器的``filter_info_topic``: filter info topic name. This need to be equal to filter_info_topic parameter。 [校准@fish]

参数 KeepoutFilter 支持的参数的完整列表列在 保留过滤器参数 页面。 [校准@fish]

重要的是要注意，只允许 KeepoutFilter 使用 global_costmap 会导致planner绕过禁区建立计划。只为 local_costmap 启用 KeepoutFilter 会导致机器人不进入禁区，但路径可能仍然会穿过它们。所以，最佳实践是启用 KeepoutFilter 全球和当地costmaps同时添加它在 global_costmap 和 local_costmap 在 nav2_params.yaml 。然而，这并不总是真的。在某些情况下，全局和本地成本图的保留区域不必相同，例如，如果机器人不允许故意进入保留区域，但如果它在那里，如果机器人夹住一个边缘或角落，它可以快速进出。对于这种情况，不需要使用本地costmap副本的额外资源。 [校准@fish]

要为全局和本地成本图启用具有相同掩码的 KeepoutFilter ，请使用以下配置: [校准@fish]

global_costmap:
  global_costmap:
    ros__parameters:
      ...
      plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
      filters: ["keepout_filter"]
      ...
      keepout_filter:
        plugin: "nav2_costmap_2d::KeepoutFilter"
        enabled: True
        filter_info_topic: "/costmap_filter_info"
...
local_costmap:
  local_costmap:
    ros__parameters:
      ...
      plugins: ["voxel_layer", "inflation_layer"]
      filters: ["keepout_filter"]
      ...
      keepout_filter:
        plugin: "nav2_costmap_2d::KeepoutFilter"
        enabled: True
        filter_info_topic: "/costmap_filter_info"

注解

所有的成本图过滤器都应该通过一个 filters 参数来启用 -- 尽管在技术上可以包括在分层成本图本身中。这与层插件是分开的，以防止层中的干扰，尤其是膨胀层。 [校准@fish]
