调整指南

本指南旨在帮助用户调整其导航系统。虽然 配置指南 是所有Nav2参数列表的所在地，但它不包含太多关于如何使用其中最重要的参数来调整系统的颜色。本指南的目的是提供更多关于如何在首次设置之外设置系统的建议，您可以在 首次机器人安装指南 找到。这绝不会涵盖所有参数 (因此，请务必查看感兴趣的包的配置指南)，但会给出一些有用的提示和提示。 [校准@fish]

此优化指南永久工作进展。如果你看到一些洞察丢失，请向票或请求智慧你想分享。这是开放截面支持慷慨Nav2用户和维护者。请考虑支付它前进。 [校准@fish]

通货膨胀潜在领域

许多用户和生态系统导航配置文件维护者发现真的失踪的充气层。虽然你确实能够简单膨胀小半径墙壁重量防止重要碰撞，* 真实 * 值充气层创建洽势场整个地图。 [校准@fish]

一些最流行的ROS导航/Nav2的调音指南，甚至 call this out specifically ，在应用了题写的成本之后，在地图的宽度上创建一个平缓的潜在场有很大的好处在实践中很少有用户这样做。 [校准@fish]

这种习惯实际上导致NavFn、Theta * 和Smac规划器产生的路径有些次优。他们真的想寻找一个平滑的潜在区域，而不是开放的0成本空间，以便呆在空间中间，更好地处理附近移动的障碍物。在搜索算法遇到通货膨胀造成的障碍之前，它将允许搜索向自由空间加权，让planner给障碍尽可能宽的铺位。 [校准@fish]

因此维护者建议，以及所有其他费用感知搜索planner可用ROS，增加通货膨胀层费用规模桡骨充分产生平滑潜在整个地图。用非常大的开放式空间，其不错0成本地区中，大厅，过道，类似; 请创建平滑潜力提供最佳性能。 [校准@fish]

机器人足迹与半径

Nav2允许用户通过两种方式指定机器人的形状: 几何 footprint 或围绕机器人的圆的半径。在ROS (1) 中，总是指定机器人的半径近似值是非常合理的，由于全球规划算法没有使用它，并且local planners/成本图是在循环假设的情况下建立的。 [校准@fish]

然而，Nav2，我们现在有多规划控制器算法利用全SE2足迹。如果你的机器人非圆，建议你给planners和控制器实际，几何足迹的机器人。这将允许planners和控制器计划或创建轨迹其更紧密地空间。例如，如果你有很长但紧身机器人，圆形假设不允许机器人计划空间只有点超出你的机器人，由于机器人不适合纵向。 [校准@fish]

的运动学可行planners (e。g. Smac混合A *，Smac状态点阵) 将使用SE2足迹碰撞检查创建运动学可行计划，如果提供实际足迹。截至2021年12月所有控制器插件支持全足迹碰撞检查确保安全路径跟踪。如果你提供足迹的机器人，将用于确保轨迹有效建议你这样做。防止一些 "stuck robot" 情况可以很容易避免。 [校准@fish]

如果你的机器人真的是圆形的，继续使用 robot_radius 参数。非圆形机器人改用半径的三个有效原因: [校准@fish]

  • 机器人很小相对于环境 (例如遥控汽车仓库) [校准@fish]

  • 机器人的计算能力如此有限，使用SE2足迹检查会增加太多的计算负担 (例如嵌入式微处理器) [校准@fish]

  • 如果你计划使用完整planner (e。g.Θ *，Smac 2D-A *，或NavFn)，你可以继续使用圆形足迹，因为这些planners不运动学可行不会利用SE2足迹。 [校准@fish]

旋转到位行为

使用 旋转控制器 ，机器人将只需旋转之前开始跟踪完整路径。这允许开发人员调整控制器插件优化路径跟踪给你清洁旋转，开箱。 [校准@fish]

这是添加由于怪癖一些现有控制器而调整控制器任务能硬质或算法并不旋转到位时完整路径 (如果欲望的特性)。结果是尴尬，口吃，或旋转行为机器人's initial and path heading's显著分歧。给控制器更好的起点开始跟踪路径调整控制器显着容易创建更直观结果作壁上观 (一个维护者的意见)。 [校准@fish]

注意: 如果使用非完整的、运动学上可行的planner (例如Smac Hybrid-a *，Smac状态晶格)，这不是必要的行为优化。这一类planner将创建考虑到机器人开始航向的计划，不需要任何旋转行为。 [校准@fish]

这种行为最适合:

  • 可以就地旋转的机器人，如差分和全向机器人。 [校准@fish]

  • 偏好旋转时开始跟踪新路径显著不同标题比机器人电流航向或调整控制器的任务使紧身旋转困难。 [校准@fish]

规范的纯追求有利于精确的路径跟随，并且通常与运动上可行的planner之一配对 (例如态晶格、混合-A * 等) 因为已知这些路径在物理约束下是可驱动的。然而，它也可以应用于差速驱动机器人，这些机器人可以轻松地旋转以匹配任何完整的路径。如果你只是想让你的机器人沿着这条路走，而不需要任何动态的障碍避免或偏差，这是你的选择。它简单而几何，并且在急转弯时在障碍物附近 * 和 * 的情况下减慢机器人的速度。 [校准@fish]

DWB TEB都选项将跟踪路径，而且偏离路径动态障碍 (为了避免)。DWB这通过评分多轨迹一套批评者。这些轨迹也产生插件可以替换，但支持的Omni Diff机器人类型有效速度和加速度限制。这些批评插件可以选择在运行时和包含权重可能调谐创建所需的行为，如最小化路径距离，距离目标或标题，和其他行动处罚可以设计。这需要一点调整给定平台，应用，行为，但可以调整DWB做几乎任何事情。 [校准@fish]

另一方面，TEB实现了一种基于优化的方法，为障碍物存在中的路径跟踪生成一个图求解问题。TEB非常擅长与场景中的其他移动代理一起很好地处理动态情况，但是其计算成本要高得多，这使得它在很大程度上不适合较小的计算平台机器人 (例如g.最小i3并以20hz运行)。这通常开箱即用，但要调整特定行为，您可能需要修改优化引擎参数，这些参数不像DWB那样直观或植根于物理内容中，但是有相当不错的默认值。 [校准@fish]

最后，旋转垫片插件帮助TEB和DWB等插件 (以及其他插件) 在开始跟踪路径之前，将机器人旋转到新路径的方向。这允许你调整你的局部轨迹planner操作所需的行为，而不必担心能够在一个非常小的欧几里得距离上旋转角距的显著偏差。。一些控制器在进行严格调整以进行精确的路径跟踪时，其操作受到限制，并且不能非常干净地旋转到新的航向。其他控制器有一个'spiral out'的行为，因为他们的采样需要一些平移速度，防止它简单地旋转到位。这有助于缓解该问题，并使机器人在适当的位置非常平稳地旋转。 [校准@fish]

Smac规划器中的缓存障碍启发式算法 [校准@fish]

Smac的混合-A*和州晶格规划器提供了一个选择， cache_obstacle_heuristic 。这可以用于缓存启发式算法，以便在相同目标姿势的填充之间使用，这可以显著提高planner的速度 (40-300% 取决于许多因素)。障碍启发式算法被用来引导机器人进入空间中间，尊重成本，并驱动运动学上可行的搜索沿着走廊走向有效的解决方案。认为它像一个2D成本感知搜索 "prime" 的planner哪里它应该去当需要花费更多的努力完全可行搜索/SE2碰撞检查。 [校准@fish]

这是有用加快性能达到更好重新速度。然而，如果缓存这种启发式，不会更新最新信息costmap引导搜索。规划，planner仍然要使用最新费用信息碰撞检查，因此这不会影响安全路径 *。然而，引导搜索羽绒新阻止走廊或指南搜索对地区，可能有新动态障碍物，可以放慢如果整个解空间阻止。 [校准@fish]

因此，建议维护者启用此只有工作在主要静态 (e。g.不多搬东西或变化，不使用活传感器更新全球costmap等) 环境规划大型空间奇异目标。目标更改Nav2，启发式将更新最新信息，所以不是很如果更改目标经常。 [校准@fish]

Nav2 Launch选项 [校准@fish]

Nav2的启动文件是非常配置。显然任何严重应用，用户应该使用 nav2_bringup 基础导航发射系统，但应移到特定存储库用户工作。典型的事情是有 <robot_name>_nav 配置包含发射参数文件。 [校准@fish]

在 nav2_bringup 中，有一个主入口文件 tb3_simulation_launch.py 。这是用于模拟机器人的主文件，包含以下配置: [校准@fish]

  • slam : Whether or not to use AMCL or SLAM Toolbox for localization and/or mapping. Default false toAMCL。 [校准@fish]

  • map : The filepath to the map to use for navigation. Defaults to map.yaml in the package's maps/ directory. [校准@fish]

  • 包装中的``world`` : The filepath to the world file to use in simulation. Defaults to the worlds/ directory。 [校准@fish]

  • param_file : The main navigation configuration file. Defaults to nav2_params.yaml in the package's params/ directory. [校准@fish]

  • 这个``autostart`` : Whether to autostart the navigation system's lifecycle management system. Defaults to true to过渡的Nav2堆栈创建到激活状态，准备使用。 [校准@fish]

  • 这个``use_composition`` : Whether to launch each Nav2 server into individual processes or in a single composed node, to leverage savings in CPU and memory. Default true to使用单过程Nav2。 [校准@fish]

  • use_sim_time : Whether to set all the nodes to use simulation time, needed in simulation. Default true for仿真。 [校准@fish]

  • rviz_config_file : The filepath to the rviz configuration file to use. Defaults to the rviz/ directory文件。 [校准@fish]

  • use_simulator : Whether or not to start the Gazebo simulator with the Nav2 stack. Defaults to true to发射Gazebo。 [校准@fish]

  • 这个``use_robot_state_pub`` : Whether or not to start the robot state publisher to publish the robot's URDF transformations to TF2. Defaults to true to公布了机器人的TF2变换。 [校准@fish]

  • 参数：use_rviz 是否使用rviz进行可视化 默认值为： true 使用RVIZ。 [校准@fishros]

  • 这个``headless`` : Whether or not to launch the Gazebo front-end alongside the background Gazebo simulation. Defaults to true to显示Gazebo窗口。 [校准@fish]

  • 这个``namespace`` : 命名空间启动机器人，如果需要。 [校准@fish]

  • 这个``use_namespace`` : Whether or not to launch robots into this namespace. Default false and为单个机器人使用全局命名空间。 [校准@fish]

  • 这个``robot_name`` : 要发射的机器人的名字。 [校准@fish]

  • 这个``robot_sdf`` : filepath机器人的凉亭配置文件包含Gazebo插件和设置模拟机器人系统。 [校准@fish]

  • 这个``x_pose``, y_pose, z_pose, roll, pitch, yaw : 在模拟中设置机器人初始位置的参数。 [校准@fish]
